1.0 INTRODUCTION

Agent is taken from the Latin word "agree" which means to do. In a layman's understanding, agent can be seen as a person or thing that does or act on behalf of another. While in computer science and industry, they basically have almost the same definition about agent in the real world but computer science refers it as a software entity while real world or layman sees it as a person, instrument, something or any other objects.

Agent based system and application is a program of software that monitors all the activities that oes on among an environment with the help of its intelligence. It mainly controls all the activities and also makes work easier for the computer user or owners. The programmer will program it in such a way it will work accordin to the instruction given to it by the user or owner.

These applications give an account about everything that happens around the environment where the computer system that carries the application is in the absent of the owner, computer set or operator.

The concept of agency is now being broadly usedf not only as a model for computer programming units displaying a certain kind of characteristics but also in a more abstract and general way as a new metaphor for the analysis specification and implementation of complex software systems.

2.0 LITERATURE REVIEW

After researching about the agent based system and application, so many founders of books defined it in different ways both in the meanimng

David and Alan (2010), defined agent based system and application as software that acts in an environment with a body, receives information through their sensors, and act in the world through their actuators or also called effectors.

According to Wooldridge and Jennings (1995), they defined agent based system and application as a computer system that is situated in some environment that is capable of autonomous action in this environment in order to meet its design objectives.

Agent based system and application refers to a software component that acts on our behalf without authority, and that is intended to do so in our best interest (Swedish Institute of Computer Science 1999-2000)

Since Artificial Intelligence is the science and engineering of making intelligent machines, so therefore agent based system and application is the center in Artificial Intelligence that always try to make things which is intelligent. This thing is not necessarily a machine and it can be considered as agent. Therefore we can conclude that agaent based system and application is the ultimate objective of Artificial Intelligence.

Russell and Nerving (2003) contributed in the definition of agent based system and application; they said that agent based sytem and application is a software that will act on behalf of other parties (computer users). This software that is acting on behalf of the computer user needs to be intelligent enough. In this definition, they concluded that agent based system and application is an intelligent software agent.

Franklin and Graesser (1996) defined agent based system and application as a software that is capable to receive command from computer users (i.e human who want to finish some task), and can act intelligently to do those tasks which has been delegated by computer users. During the activities, an agent basically can interact with the environment to do its task without much interaction and command from computer users.

2.1 AGENTS AND REGULAR (NON-AGENTS) SOFTWARE

  1. Autonomy: Agents should be able to perform the majority of their problem solving tasks without the direct intervention of a human or agents and they should have a degree of control over their own actions and their own intenal state.

  2. Social Ability: Agents should be able to interact when they deem appropriate with other software agents and humans in other to complete their own problem solving and to help others with their activities where appropriate.

  3. Responsiveness: Agents should persive their environment (which may be the physical world, a user, a collection of agents, the internet etc) and response in a timely fashion to change which occur on it.

  4. Proactiveness: Agents should not simply act in response to their environment, they should be able to exhibit appropriate goal directed behaviour and take the initiative where appropriate.

2.2 ADVANTAGES OF AGENT BASED SYSTEM AND APPLICATION

2.2.1 New Real Time Continuously Monitoring: The end point agent resident in a computer asset can monitor, secure configuration parameters continuously or periodically and report compliance to the repository sub-system (database).

2.2.2 Complex Management Tasking: The agent is a mission application; it may be programmed to perform complex management tasks such as checking the host platform on ti security configuration parameters and enforcing the installation of security patches.

2.2.3 Deep Visibility to Security Configuration: Because the agent resides locally on computing assist and is executed as a system process, it usually has direction access to the security kernel; it has visibility to priviledge system configuration files.

2.3 DISADVANTAGES OF AGENT BASED SYSTEM AND APPLICATION

Agent based system and application has some challenges that have hindered people from using it, they are;

2.3.1 Higher Operation Costs: End point agents are required to be installed on target computer assets. For a large enterprise, this entails greater operation costs installation and maintenance (i.e patch upgrade).

2.3.2 Incompatibility Between Agents: An end point agent is a mission specific application that runs on top of an operation (0/5). A host may run multiple agents, such as one for antivirus systems, one for instruction detection (check point zone alarm), and another one for configuration management (Tivolic configuration manager). Introduction of an endpoint agent may disrupt or interfere with existing agents.

2.3.3 Endpoint Agents Require Configuration Management: The agent is a software and so it may require a software distribution services to manage configurations. Therefore the repository sub system (database) requires an agent distribution program to track and maintain agent configuration throughout an enterprise.

2.3.4 Not All Application Domains are Appropriate for Agent: Application domains which are represented by application domain object provides isolation, uploading and security boundaries. this can be barriers for agents.

2.4 SOME APPLICATION DOMAINS WHERE AGENT TECHNOLOGIES WILL PLAY A CRUCIAL ROLE

  1. Ambient Intelligence: The seamless deliveries of ubiquitous computing, continuous communication and intelligent user interfaces to consumer and industrial devices.

  2. Electronic Business: Where agent based approaches are already supporting the automation and semi-automation of information gathering activities and purchase transaction over the internet.

  3. Semantic Web: Where agents are neede both to provide services and to make best use of the resources available, often in co-operation with others.

  4. Bio Information and Computational Biology: Where intelligent agents may support the coherent exploration of data revolution occuring in biology.

  5. Grid Computing: Where multi-agent system approaches will enable efficient use of the resources of high performance computing infrastructure in science, engineering, medical and comercial application.

3.0 EXPLANATION OF AGENT BASED SYSTEM AND APPLICATION

Early, Artificial Intelligence (A.I) papers predicted the development of complete system that would exhibit most of the characteristics that will associate with intelligence in human behaviour. Such system will perceive the environment through artificial eyes and ears and would communicate in natural languages, reason is to solve problem and formulate plans that will also act on the formula to archieve their goals and would learn from their experience. Artificial intelligent researchers have seen that it is difficult to develop, then use the features or study the function of an intelligent system, then brought to a conclusion that artificial intelligent building system exhibits several cognitive functions associated with human intelligence is now feasible and such system are called Agent Based System (ABS).

Many artificial intelligent researchers like Russell and Nerving (2013); Wooldridge and Jennings (1995); Smith et al (1994); Franklin and Graesser (1996), brought to general characterization of an agent based system and application. They defined it as a knowledge based system that perceives its environment (which may be the physical world, a user via a graphical user interface, a collection of other agents, the internet or other complex environment), reasons to interpret perceptions, draw inference, solve problems and determine actions and upon that environment to realize a set of goals or tasks for which it was designed.

It can accept high level requests indicating what the user want and can decide how to satisfy each request with some degree of independence or autonomy exhibiting goal, with directed behaviour, dynamically choosing which to take, and in what sequence it can collaborate with its users to improve the accomplishment of the tasks or can carry out such tasks on users behalf. In so doing, it can employ some knowledge or representation of the user, advise the users on how to perform a task, train the user or can also help different users to collaborate. The behaviour of the agent is based on a correspondence between the external application domain such as object and classes of objects, laws and actions. The interference engine consist of the programs that manipulate the data structure in the knowledge base in order to solve the problems for which the agents were designed.

3.1 CHARACTERISTICS OF AN AGENT BASED SYSTEM AND APPLICATION

The following are the characteristics of an agent based system and application:

  • It has some attribute of a real human.
  • It represents the user in the absent of the computer user.
  • It takes, retieve, store and deliver information to the user.
  • It is an intelligent program that pursues its own goals, interacts with other agents and perceives the world/environment.

3.2 AGENTS ARE GOOD FOR SYSTEM THAT;

  1. Have distributed components.
  2. Are distributed over a network.
  3. Exhibits dynamic changes.
  4. Handle large quantities of information for multiple heterogeneous sources.
  5. Require autonomous behaviour.
